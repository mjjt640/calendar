# 重复日程配置流设计

## 目标

在现有 Java Android 日程表项目中补齐“重复日程配置页 + 新建/编辑页联动”能力，确保：

- 新建页在开始时间和结束时间都有效后，出现“重复与持续时间”卡片。
- 用户可进入独立的重复配置页编辑 `RecurrenceDraft`。
- 编辑普通日程时可以进入同一配置页，将其转换为重复日程。
- 编辑重复日程时，先弹出居中的范围选择弹窗，再进入配置页。
- 配置页返回后，主页面摘要即时刷新，但真正写库仍由保存动作统一触发。

## 本轮范围

### 本轮实现

- 新增 `RecurrenceConfigActivity`
- 让 `AddScheduleViewModel` 持有并管理 `RecurrenceDraft`
- 在新建/编辑页显示重复卡片和摘要
- 编辑重复日程时增加中间确认弹窗
- 配置页返回后回填摘要与草稿
- 为新增状态流补齐 ViewModel 单元测试

### 本轮不做

- 删除重复日程时的范围弹窗
- 更复杂的重复规则扩展
- 月历页和首页列表里更深的重复交互优化

## 方案选择

采用“独立配置页 + ViewModel 主导”的方案。

原因：

- 与现有 `AddScheduleActivity` 结构兼容，接入成本可控。
- 业务边界清晰：主页面负责整体草稿，配置页只负责编辑重复规则。
- 后续扩展删除范围弹窗、拆分系列写库时，状态模型可复用。

不采用“只做假入口”的原因是会留下 UI 完成但业务未接通的半成品；不采用“Activity 主导拼装所有状态”的原因是后续维护和测试成本更高。

## 页面与交互流

### 新建页 / 编辑页

- 保留现有基础信息卡片和时间卡片。
- 当开始时间、结束时间都有效时，显示“重复与持续时间”卡片。
- 该卡片默认展示“单次”，配置完成后展示摘要，例如：
  - 每天
  - 每周
  - 每 2 周，共 12 次
  - 每月，截止到 2026-06-30
- 点击卡片后进入 `RecurrenceConfigActivity`。
- 返回后仅刷新卡片摘要，不直接落库。

### 重复配置页

- 顶部使用中文标题栏和返回按钮。
- 页面主体分为三块：
  - 重复频率
  - 重复间隔
  - 结束条件
- 选择“自定义”时显示间隔设置。
- 选择“截止日期”或“共几次”时显示对应输入区域。
- 页面底部固定主按钮“保存设置”。
- 点击保存后，通过 `Activity Result` 回传 `RecurrenceDraft`。

### 编辑重复日程时的范围选择

- 如果当前日程不是重复日程，点击重复卡片后直接进入配置页。
- 如果当前日程是重复日程，先弹出居中的 Material 对话框。
- 对话框提供两个主选项：
  - 仅这一次
  - 这一次及之后
- 用户选定范围后，再进入配置页。
- 这一步只覆盖“编辑重复配置”的范围选择，不包含删除。

## 动效要求

- 新建页中的重复卡片首次出现时，使用淡入 + 轻微上移动效。
- 主页面进入配置页时，采用右进左出的页面转场。
- 配置页返回时，采用反向转场。
- 摘要刷新后，卡片摘要区域做一次轻量高亮过渡。

动效目标是“顺滑、有层次”，但不要夸张，不破坏当前偏中国本地产品的视觉节奏。

## 状态设计

### AddScheduleViewModel

`AddScheduleViewModel` 继续作为新建页和编辑页的主草稿容器，除现有字段外新增：

- `RecurrenceDraft currentRecurrenceDraft`
- 重复卡片是否显示
- 重复摘要文本
- 当前是否为重复日程
- 当前编辑范围 `OccurrenceEditScope`

职责：

- 持有主页面完整草稿
- 计算主页面展示用摘要
- 接收配置页回传结果并刷新 UI 状态
- 在最终保存时决定走普通保存还是重复保存分支

### RecurrenceConfigViewModel

`RecurrenceConfigViewModel` 只负责重复规则编辑，不直接调用仓库。

输入：

- 当前 `RecurrenceDraft`
- 当前开始时间
- 当前结束时间
- 可选编辑范围 `OccurrenceEditScope`

输出：

- 校验后的 `RecurrenceDraft`
- 配置页内部摘要
- 各配置块显隐状态
- 保存按钮可用状态

职责边界：

- 配置页内部即时反馈由该 ViewModel 管理
- 主页面最终展示摘要由 `AddScheduleViewModel` 统一生成

## 数据与保存策略

### 新建日程

- 未配置重复规则时，沿用现有普通日程保存流程。
- 配置了重复规则时：
  - 先保存基础日程
  - 再写入对应的 `recurrence_series`

### 编辑普通日程

- 支持将普通日程转换为重复日程。
- 保存时更新基础日程，并补写 `recurrence_series`。

### 编辑重复日程

- 进入配置前先选择范围。
- `仅这一次`
  - 保存时走单次 override exception 分支
- `这一次及之后`
  - 保存时走系列拆分 / 截断 + 新系列分支

本轮重点是把 UI、状态流和保存入口接通，底层依赖前面已完成的数据层与窗口解析能力。

## Activity 与 ViewModel 分工

### AddScheduleActivity

负责：

- 绑定现有表单控件
- 观察 ViewModel 状态并更新界面
- 控制重复卡片的显示、点击和过渡动画
- 在需要时弹出“编辑范围选择”中间对话框
- 使用 `Activity Result` 打开配置页并接收返回结果

不负责：

- 拼装重复摘要
- 判断重复规则是否合法
- 直接操作仓库中的重复规则数据

### RecurrenceConfigActivity

负责：

- 呈现重复频率、间隔、结束条件 UI
- 观察 `RecurrenceConfigViewModel`
- 返回用户确认后的 `RecurrenceDraft`

### ViewModel 边界

- Activity 只负责界面交互和事件转发
- 业务状态、摘要文案、保存分支判断全部收口到 ViewModel

## 测试策略

### AddScheduleViewModelTest

补充以下测试：

- 开始/结束时间有效时显示重复卡片
- 配置回写后摘要正确刷新
- 普通日程保存仍保持原有行为
- 带重复草稿的新建保存进入重复保存分支

### RecurrenceConfigViewModelTest

补充以下测试：

- 默认状态为单次
- 每天 / 每周 / 每月 / 自定义切换时摘要正确
- 自定义间隔为 0 时禁止保存
- 结束条件切换时显隐状态正确

### 手工验证

- 新建页选择开始和结束时间后，重复卡片正常出现
- 点击卡片可进入配置页
- 配置后返回，摘要即时刷新
- 编辑普通日程时可以进入配置页
- 编辑重复日程时先出现中间范围弹窗
- 选择任一范围后可以继续进入配置页
- 不配置重复时，原有新建和编辑流程不回归

## 兼容性与风险控制

- 不改动现有普通日程的默认保存逻辑，减少回归风险。
- 不在本轮同时接入删除范围弹窗，避免范围膨胀。
- 已存在的乱码规格不复用，改为新建 UTF-8 文档作为后续实现依据。
- 现有字符串资源存在历史编码问题，本轮只为新功能补必要项，不扩大清理范围。

## 实施顺序

1. 补规格文档并确认
2. 新增 `RecurrenceConfigActivity` 与对应布局
3. 扩展 `AddScheduleViewModel` 的重复草稿状态
4. 在 `AddScheduleActivity` 接入重复卡片、页面跳转和中间弹窗
5. 回填摘要并接通最终保存链路
6. 补充和修正对应单元测试

## 完成判定

满足以下条件即可认为本轮功能完成：

- 新建页和编辑页都能看到并使用重复卡片
- 配置页能返回有效草稿
- 编辑重复日程时会先出现中间范围弹窗
- 摘要能正确刷新
- 普通日程流程不回归
- 相关 ViewModel 单元测试通过
