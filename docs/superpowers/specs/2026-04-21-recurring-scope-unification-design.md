# 重复日程范围统一设计

## 目标

统一重复日程的编辑、删除范围能力，支持：

- 仅这一次
- 这一次及之后
- 整个系列

本轮覆盖编辑与删除，不扩展到提醒、首页或备份模块。

## 规则

### 范围枚举

`OccurrenceEditScope` 扩展为：

- `SINGLE`
- `THIS_AND_FUTURE`
- `ENTIRE_SERIES`

### 编辑语义

- `仅这一次`
  - 只改普通字段：写 override exception
  - 如果修改重复规则：当前这次脱离原系列，保存为新的独立事项；原系列当前次记为已替换/跳过
- `这一次及之后`
  - 旧系列在当前次之前截断
  - 从当前次开始创建新系列
  - 即使只改普通字段，也按拆分后半段系列处理
- `整个系列`
  - 直接更新整个系列的锚点事项与重复规则

### 删除语义

- `仅删除这一次`
  - 写取消型 exception，当前次不再显示
- `删除这一次及之后`
  - 将系列截断到当前次之前
- `删除整个系列`
  - 删除整个系列锚点与 series 关联

## 分层

- `Activity`
  - 负责弹窗、收集范围、展示结果
- `ViewModel`
  - 负责保存草稿、转发动作为统一请求
- `Repository`
  - 负责执行范围语义，不让 UI 层拼业务

## UI

### 编辑弹窗

- 修改这一次
- 修改这一次及之后
- 修改整个系列

### 删除弹窗

- 仅删除这一次
- 删除这一次及之后
- 删除整个系列

普通单次事项保持原来的直接编辑、直接删除流程。

## 实现顺序

1. 统一范围枚举与仓库入口
2. 接入编辑链路
3. 接入删除链路
4. 补测试与回归验证

## 测试

- Repository：
  - 编辑三种范围
  - 删除三种范围
- ViewModel：
  - 范围选择状态
  - 保存分支
  - 删除分支
- 回归：
  - 普通事项流程不变
  - 现有重复配置流不回归

## 完成判定

满足以下条件即完成：

- 编辑重复日程支持三种范围
- 删除重复日程支持三种范围
- `仅这一次` 改重复规则时会转为独立事项
- 普通事项不受影响
- 相关单测通过
